# 🎯 앱 미션 — RuleKeeper (룰키퍼)

> "규칙을 지키게 해주는 개인용 시그널 엔진" · 폴더명 `trading-signal-engine`
> 프로젝트 이름은 임시 제안 — 바꿔도 됩니다.

## 미션 문장
> 재량으로 매매하던 트레이더가 자신의 진입 규칙을 **명시적 신호**로 바꿔, 심리에 흔들리지 않고 규칙대로 매매하며 그 **준수 여부와 성과를 숫자로 확인**하고, 그 데이터로 규칙을 스스로 다듬어 가게 해주는 1인용 규칙 기반 시그널 엔진.

## 문제 정의
지금은 재량매매라 진입 근거가 머릿속에만 있고 말(규칙)로 표현되지 않은 상태다. 그래서 (1) 왜 들어갔는지 근거가 매번 흔들리고, (2) 트레이딩 중 심리에 휘둘려 규칙 아닌 충동으로 매매가 어긋난다. 규칙을 밖으로 꺼내 신호로 만들면, 따를지 말지를 스스로 선택하고 그 선택의 결과를 데이터로 되돌아볼 수 있게 된다.

- **v1 대상 (모두 비트겟, 동일 API):** BTC(`BTCUSDT`) · ETH(`ETHUSDT`) · 나스닥100(`NDX100USDT`, RWA 무기한)
- **v1 체결은 3개 전부 비트겟**(나스닥 = `NDX100USDT`) — 데이터·신호·체결·일지를 한 거래소로 통일. HFM(MT5) NQ 체결 연동은 v2 옵션.

> **데이터 근간 결정(웹 검증됨, 2026-07):** 지지·저항을 TradingView/비트겟 지표에서 "가져오는" 건 불가능/비현실적이다.

## v1 규칙 엔진 스펙 (이 앱의 심장)

사용자는 스스로를 "추세매매자"라 하지만, 실제로 만족스러웠던 셋업은 *"저점을 안 깨는 지지에서 매수 / 고점을 안 깨는 저항에서 매도 후 한 구간 익절"* 이었다. 이 둘은 모순이 아니라 하나로 통합된다 → **상위 TF 추세 방향으로, 되돌림(pullback) 시 지지·저항 레벨에서 진입하는 "추세-되돌림 매매"**.

신호는 아래 **4개 층을 모두 통과**해야 발생한다.

### ① 방향(Bias) 판정 — 상위 TF (4H/1H)
- **HH/HL 시장구조**(연속 고점·저점이 높아짐/낮아짐, 스윙 피벗으로 탐지) **AND**
- **EMA 정배열**(상승: 가격 > EMA50 > EMA200 / 하락: 대칭)
- 두 조건이 **동시에 같은 방향**일 때만 "추세 명확"으로 인정. (교차검증)

### ② 횡보 게이트 — "횡보면 진입 금지" (사용자 페인포인트 직접 해소)
- **ADX > 25** 일 때만 진입 허용. (20~25 관망, **< 20 진입 금지**)
- 구조와 이평이 상충하거나 ADX 미달이면 → `range → no entry`.
- 사용자가 "횡보 구간이 어렵다"고 한 문제를 숫자 게이트로 차단하는 핵심 장치.

### ③ 진입존(S/R) 자동 탐지 — 되돌림 지점
- **스윙 피벗(프랙탈) 탐지 + 근접 레벨 병합**으로 S/R "존" 생성. (OHLCV만으로 계산)
- **레벨 강도 = 방어 횟수**로 가중. (여러 번 지켜진 존일수록 신뢰↑ — 사용자 의견 반영: "캔들이 여러 번 지지한 곳을 엔진이 객관적으로 더 신뢰")
- 저비용 컨플루언스로 **전일·전주 고저 + 라운드 넘버** 추가.
- (v1 제외) 수동 레벨 오버라이드 — 사용자가 "임의로 그으면 유의미성이 불확실"하다고 판단해 v1에선 자동탐지만. (v2 옵션)

### ④ 진입 트리거 — 하위 TF (15m) + 거래량 확증
- **되돌림 확인**: 가격이 지지존을 **아래꼬리로 테스트**한 뒤, **15m 종가가 다시 그 존 위(추세방향)로 마감**하면 "레벨 방어 확인". (하락추세는 저항존에서 대칭)
- 그 캔들이 **반전형(아래꼬리 > 몸통)**이면 가중치↑.
- **거래량 확증(소프트 가중치)**: 방어 캔들의 거래량이 **최근 20봉 평균 대비 ≥ 1.3×**면 "강한 신호", 미만이면 "약한 신호"로 **라벨링**. → 하드필터가 아니라 기록만 하고, 피드백 루프가 "고거래량 방어일 때 실제 승률이 높은지"를 측정하게 함. 검증되면 v2에서 하드필터로 승격.
- 스타일: 데이 또는 스윙, **하루 최대 1회 진입**을 기본 리듬으로.

### ⑤ 손절 — "위치"와 "크기"를 분리 (리스크 교정)
사용자가 말한 "총 시드의 10% 이내"는 1회 매매 리스크로 과도(연패 시 계좌 파괴적). 분리한다:
- **손절 위치**: 구조 기준 — 직전 스윙 아래(롱)/위(숏), 또는 방어한 레벨 바로 바깥.
- **손절 크기(계좌 리스크)**: 계좌의 **고정 1% 리스크**. (0.5~1% 중 확정값 = **1%**, 나중에 다이얼로 조절)
- **포지션 크기 = (계좌 잔고 × 1%) ÷ 손절 거리.** → 손절 폭이 넓어도 계좌 리스크는 항상 일정.

### ⑥ 익절 / 트레일링 — 하이브리드
- **1차 목표**: **+2R** 도달 시 부분(또는 전량) 익절.
- **남은 물량**: **구조 기반 트레일** — 15m에서 새 스윙이 생길 때마다 스탑을 직전 스윙 아래로 이동.
- 대안(참고): ATR 트레일 / 샹들리에 엑싯 — v1은 위 하이브리드 기본.

### ⑦ 멀티 심볼 리스크 상한 (상관 그룹 합산)
여러 심볼을 동시에 다루면 상관 포지션이 리스크를 몰래 증폭한다.
- **`{BTC, ETH}`는 고상관 → 한 묶음으로 합산 리스크 상한**(예: 동시 진행 시 합산 ≤ 1.5%, 각 1%가 아니라).
- **나스닥(`NDX100USDT`)은 별도 그룹**(크립토와 부분 상관) — 자체 상한.
- 하루 최대 진입/동시 포지션 수 상한도 함께. → 3개 신호가 한꺼번에 떠도 계좌가 감당 못 할 리스크를 지지 않게.

## 🔄 피드백 루프 / 데이터 축적 (이 앱의 진짜 가치)

신호는 **시작점**일 뿐, 가치는 "신호 → 내 반응 → 결과 → 학습"의 닫힌 고리에서 나온다.

```
① 엔진이 신호 발생 → ② 텔레그램으로 진입근거 카드 전송
                          ├─ [✅ 진입함] [❌ 건너뜀] [⏭ 관망]  ← 사용자 탭 (= "신호 준수율"의 원천)
                          ▼
③ 결과 기록 (익절 +2R / 손절 -1R / 수동청산 몇 R)  ← v1은 텔레그램 수동 입력(A안)
                          ▼
④ 매매일지에 한 줄 자동 축적 [신호조건 전부 + 진입/건너뜀 + 이유 + 결과 R]
                          ▼
⑤ 주간/월간 리포트로 피드백
```

**텔레그램 진입근거 카드 예시:**
```
🟢 롱 신호 · BTCUSDT
진입존: 63,200~63,450 (지지, 4회 방어)
추세: 4H 상승 (HH/HL ✓, EMA정배열 ✓, ADX 28)
트리거: 15m 종가 존 위 마감 ✓ (반전 캔들, 거래량 1.5× → 강)
손절: 62,900 (-1R / 계좌 1%)   목표: +2R → 64,400
[✅ 진입함]  [❌ 건너뜀]  [⏭ 관망]
```

**결과 기록 방식(확정):** (A) **텔레그램 탭 + 청산가 수동 입력.** API 키 불필요, 가장 단순. → 이후 (B) 비트겟 읽기전용 API로 실제 체결을 자동 대조(타이핑 0)하는 v1.5 업그레이드로 확장.

**리포트가 돌려주는 것:**
- 신호 준수율(%), 준수 vs 이탈 성과 격차("건너뛴 신호가 알고 보니 +R이었나")
- 셋업 승률 / 평균 R
- **조건별 성과**(ADX 구간별·방어횟수별·거래량 강/약별 승률) ← 규칙을 스스로 다듬는 재료이자 v2 복기 에이전트가 먹을 데이터

## 핵심 기능 (v1) — 5개
1. **규칙 기반 진입 신호 생성 + 텔레그램 알림** — 4개 층 통과 시 "지금 신호 떴다".
2. **진입 근거 카드** — 어떤 조건(추세·ADX·레벨 방어횟수·캔들 마감·거래량)이 충족됐는지 표시.
3. **자동 매매일지 로깅** — 신호 / 실행 여부(따름·어김·이유) / 결과(R)를 기록.
4. **성공지표 자동 계산 대시보드** — 신호 준수율·셋업 승률·평균 R·조건별 성과.
5. **가벼운 심리 가드레일** — 일일 손실 한도 경고 + 리벤지/과매매 경고.

## 안티스코프 (v1에서 하지 않는 것 → v2 로드맵)
- **자동 체결(봇 직접 주문)** — v1 체결은 사용자가 직접.
- **비트겟 체결 자동 대조(B안)** — v1.5.
- **백테스트 엔진** — v2. (NQ는 CFD라 스프레드·스왑·체결 차이로 백테스트가 낙관적으로 나오는 문제 있음)
- **HFM/MT5 어댑터** — v2. (나스닥 신호 자체는 v1에서 비트겟 `NDX100USDT`로 처리; HFM CFD 체결 연동만 v2)
- **비트겟 500x `NAS100` CFD 라인** — 사용 안 함(별도 상품, mix API 미지원).
- **무거운 거래량 분석** — 볼륨 프로파일(HVN/POC), 미결제약정(OI)·펀딩비·CVD·테이커 델타 등 크립토 전용 신호는 v2.
- **수동 레벨 오버라이드** — v2 옵션.
- **복기·패턴분석 멀티에이전트(전략·리스크·복기)** — v2. v1은 그 재료가 될 일지 데이터 축적까지만.
- **다중 사용자 / 신호 공유·판매** — 범위 밖(현재 1인용).

## 성공 지표 (측정 가능)
목표 수치 칸은 예시 — 사용자가 자신에게 맞는 목표로 채워 주세요.

| 지표 | 목표(예시) | 기간 |
|------|-----------|------|
| 신호 준수율 (뜬 신호 중 규칙대로 실행한 비율) | ≥ 80% | v1 가동 후 4주 |
| 셋업 승률 (신호대로 실행한 매매의 승률) | 채워주세요 (예: ≥ 45%) | 표본 30회 이상 |
| 평균 손익비 R (실현 손익 ÷ 리스크) | ≥ 1.5R | 표본 30회 이상 |
| (심리) 일일 손실 한도 위반 횟수 | 월 0~1회 | 매월 |
| (심리) 리벤지/과매매 경고 발생 횟수 | 감소 추세 | 매월 |

> 초기에는 승률(느리고 노이즈 큼)보다 **신호 준수율**과 **준수 시 R**이 두 페인포인트(근거 명확화·심리 안정)를 더 잘 반영한다.

## 구조 메모 (확장성 관점)
> 기술 스택 결정이 아니라, 나중에 NQ로 확장될 수 있게 하는 구조 방향. 상세 기술 선택은 dev-kickstart 단계에서.
- **플랫폼 독립 "핵심 규칙 엔진"**(추세-되돌림 셋업 로직) + **심볼별로 도는 멀티 심볼 러너** + **거래소별 어댑터** 구조.
- **v1: Bitget 어댑터** 하나로 `BTCUSDT`/`ETHUSDT`/`NDX100USDT` 모두 처리 (캔들 수신 → 규칙 판정 → 텔레그램 알림 → 일지 로깅 → 리포트). 나스닥 심볼만 세션·갭 처리 분기.
- **v2: MT5 어댑터**로 HFM NQ(CFD)를 동일 규칙 엔진에 연결. (CFD 특성상 검증 단계에서 스프레드·스왑·체결 차이 반영 필요)
