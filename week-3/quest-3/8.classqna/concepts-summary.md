## 📍 1주차 개념

### AI
- **AI ≠ 신, AI = Context 연산기계**
- **Transformer (2017, Google) ≒ 귀납법**: 인터넷의 수많은 문장을 학습하며 "오늘 날씨가 ___" 뒤에 올 말의 패턴을 스스로 깨우침. 누가 규칙을 알려준 게 아니라 수억 개 사례에서 법칙을 만들어냄(귀납). 외국에서 현지인 말을 수천 번 듣고 문법책 없이 감으로 익히는 것과 비슷.
- **Thinking (Reasoning / 추론) ≒ 연역법**: "17 × 23은?"을 일반 모드는 바로 답하지만, Thinking 모드는 "곱셈은 이렇게 푼다 → 17을 분해 → 각각 계산 → 따라서 391" 처럼 법칙을 하나씩 적용(연역).

### Context
- **Context = 문맥/맥락/상황** — 어떤 말·사건이 일어난 배경. "그 발언은 context를 알아야 이해할 수 있다."
- **Context 만들기 3가지**: ① 우리가 주입 ② AI가 스스로 찾게 함 ③ Rules로 정해둠
- **Text Context**: 컴퓨터는 사실 모두 텍스트(0과 1) → Markdown 파일, Code
- **Image Context (Multi Modal)**: 이미지를 Patch(작은 조각)로 나눠 벡터화 & Self-Attention → 텍스트와 이미지를 같은 아키텍처로 처리
- **MCP (Model Context Protocol)**: Figma, Chrome Dev Tool, CLI 등. AI가 잘 이해·작동하는 '텍스트 기반' 인터페이스 (GUI는 아직 어려워함)

### Git & GitHub
- **Git**: 무료·오픈소스 분산 버전 관리 시스템. 작은~대형 프로젝트까지 빠르고 효율적으로 관리. 확인: `git --version` (https://git-scm.com)
- **GitHub**: 수백만 개발자가 쓰는 AI 기반 개발자 플랫폼. 로그인: `gh auth login` (https://github.com)

## 📍 2주차 개념

### 앱의 구조 (Web Application)
- (상세 본문은 이미지 다이어그램으로 제공되어 텍스트 추출 불가 — "AI Factory Manager / Web Application" 다이어그램)

### Browser: HTML, CSS, JS
- 핵심 메시지: **HTML은 구조(뼈대), CSS는 꾸미기(색/크기/모양), JavaScript는 변화(움직임/상태)**
- **HTML**: 화면에 "무엇이 있는지" 정함 — 예: `<div id="box">Hello</div>`
- **CSS (Tailwind)**: "어떻게 보일지" 정함 — 색/크기/둥근 모서리/그림자 등 (정적, 멈춰있는 모습)
- **JavaScript**: 클릭하면 클래스/텍스트를 바꿔 "변화"를 만듦 — 버튼으로 이모지를 춤추게 하는 예제(`setInterval`로 `style.transform`을 계속 변경)
- Cursor로 `index.html` + `client.js`를 직접 작성해보는 실습 포함

### 앱의 10가지 유형
**핵심 기능 (앱의 정체성)**
1. 계산기 (Calculation) — 입력 → 공식 → 새 정보 (BMI, D-day)
2. 저장소 (DB/Storage) — 데이터 저장·조회 (식단 기록, 단어장)
3. 중계기 (Network) — 외부 서버 데이터를 가져와 보여줌 (날씨앱, 포켓몬 도감)
4. 감지/제어기 (Sensor & Control) — 물리 세계와 소통 (GPS, 소음 측정, IoT)
5. 변환기 (Transform) — 같은 정보를 다른 형태로 (QR 생성, 단위 변환)
6. AI (Intelligence) — AI가 판단/생성 (AI 상담소, 닉네임 생성기)

**인프라 기능 (앱을 지탱하는 기반)**
7. 자동화기 (Automation) — 앱이 먼저 행동 (물 마시기 리마인더)
8. 연결기 (Communication) — 사람과 사람을 실시간 연결 (채팅방)
9. 문지기 (Auth) — 신원 확인·권한 통제 (로그인)
10. 거래기 (Payment) — 돈이 오가는 거래 (결제, 후원)

**실제 앱 = 조합** (핵심 1~2개 + 필요한 인프라)
- 카카오톡 = 연결기 + 저장소 (+ 자동화기 + 문지기)
- 배달의민족 = 중계기 + 감지/제어기 (+ 저장소·자동화기·문지기·거래기)
- ChatGPT = AI + 저장소 (+ 변환기·문지기·거래기)

**학습 순서**: ① 계산기 → ② 변환기 → ③ 저장소 → ④ 중계기 → ⑤ AI → ⑥ 문지기 → ⑦ 자동화기 → ⑧ 연결기 → ⑨ 감지/제어기 → ⑩ 거래기
(프론트만 → 서버 도입 → API 호출 → 인프라 → 심화)

### 앱의 유형: 계산기 (Calculation)
- **정의**: 입력값을 정해진 규칙/공식으로 새로운 정보를 만들어냄. 방향: 입력 → 앱 → 결과
- **특징**: 같은 입력 → 항상 같은 결과(결정론적). 저장 안 함, 외부에 묻지 않음
- **예제**: BMI 측정기(키+몸무게→BMI 판정), 오늘의 운세(랜덤 선택), D-day 계산기(날짜 차이)
- **구분 포인트**: "가진 데이터만으로 바로 답이 나오는가?" → 계산기. **변환기와 차이**: 계산기는 *새로운 정보*를 만들고, 변환기는 *같은 정보*를 다른 형태로 바꿈

### 앱의 유형: 변환기 (Transform)
- **정의**: 입력을 다른 형태/포맷으로 바꿔줌. 방향: A형태 → 앱 → B형태
- **특징**: "답"을 구하는 게 아니라 같은 정보를 다른 모습으로. 되돌리면 원본이 나옴
- **예제**: QR 코드 생성기(텍스트→QR), 마크다운 미리보기(MD→HTML), 단위 변환기(cm↔inch)
- **구분 포인트**: "입력·출력이 같은 정보인데 형태만 다른가?" → 변환기. 쉬운 구분: **원본 복원 가능하면 변환, 새 값이 나오면 계산**
