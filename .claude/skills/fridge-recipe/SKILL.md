# 냉장고 레시피 추천 (fridge-recipe)

냉장고에 있는 재료를 담은 `.json` 파일을 읽어, 그 재료들을 조합해 **실제로 만들 수 있는 요리**를 추천하는 스킬입니다.

## 재료 데이터 위치

기본 재료 파일:
- `week-4/quest-4/ingredients-skill/ingredients.json`

같은 폴더에 여러 개의 `.json`이 있으면 **모두 읽어서** 재료를 합칩니다. 사용자가 특정 파일 경로를 주면 그 파일을 우선합니다.

각 재료 객체 형식:
```json
{ "name": "계란", "quantity": "7개", "category": "냉장" }
```

## 작업 흐름

1. **재료 읽기** — 위 경로(또는 사용자가 지정한 경로)의 `.json` 파일을 Read 도구로 읽습니다. 폴더에 파일이 여러 개면 Glob으로 `*.json`을 찾아 전부 읽습니다.
2. **재료 목록 정리** — `name` / `quantity` 를 기준으로 가진 재료를 표로 정리합니다. **JSON에 실제로 적힌 재료만** 사용하고, 없는 재료를 있다고 가정하지 않습니다.
3. **레시피 2~3개 추천** — JSON에 있는 재료만으로 만들 수 있는 요리를 2~3가지 제안합니다. 각 레시피마다:
   - **요리 이름**
   - **사용하는 냉장고 재료** (주재료는 JSON에서 나온 것 + 기본 양념은 `(양념)` 표시)
   - **간단 조리법** (3~5단계)
   - **예상 소요 시간**
4. **레시피를 md 파일로 저장** — 추천한 각 레시피를 반드시 마크다운 파일로 저장합니다. (아래 "결과물 저장" 참고)

## 레시피 작성 규칙 (반드시 지킬 것)

- **⚠️ 재료 제한 — 가장 중요:** 레시피의 **주재료**는 반드시 **JSON 파일에 실제로 있는 재료**여야 합니다.
  - **예외 — 기본 양념은 허용:** 물, 소금, 후추, 식용유, 참기름, 간장, 설탕, 고춧가루, 다진 마늘 등 자취방에 흔히 있는 **기본 양념·조미료**는 JSON에 없어도 사용할 수 있습니다.
  - 단, **주재료(고기·채소·달걀·두부 등 요리의 핵심 재료)는 JSON에 있는 것만** 씁니다. JSON에 없는 새 주재료(예: 감자, 양파, 소고기)를 임의로 추가하지 않습니다.
  - 기본 양념으로 쓴 재료는 레시피에서 `(양념)` 으로 표시해 JSON 재료와 구분합니다.
  - 가진 재료로 만들 수 있는 요리가 마땅치 않으면, 억지로 지어내지 말고 "현재 재료만으로는 어렵다"고 솔직히 말합니다.
- `week-1/recipe/된장찌개.md` 의 형식을 참고합니다 (제목 → 재료 → 만드는 법 → 팁).
- 자취생 기준: 조리 도구는 냄비·프라이팬·전자레인지 정도로 가정하고, 20분 내외의 간단한 요리를 우선합니다.
- 가진 재료를 **최대한 많이** 쓰는 조합을 우선합니다.
- 추천은 한국어로, 친근하고 간결하게 작성합니다.

## 결과물 저장 (필수)

스킬을 실행하면 추천한 레시피를 **항상 마크다운 파일로 저장**합니다. (사용자가 따로 요청하지 않아도 저장)

- 저장 위치: `week-4/quest-4/ingredients-skill/recipes/<요리이름>.md`
- 형식: `week-1/recipe/된장찌개.md` 와 동일 (제목 → ## 재료 → ## 만드는 법 → ## 팁)
- 레시피가 여러 개면 각각 별도 `.md` 파일로 저장합니다.
- `## 재료` 항목에는 **JSON에서 온 주재료**와 **기본 양념**을 함께 적되, 양념은 `(양념)` 으로 표시합니다.
- 저장 후 어떤 파일이 만들어졌는지 사용자에게 알려줍니다.
