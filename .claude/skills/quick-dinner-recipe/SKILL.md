# 15분 저녁 레시피 (quick-dinner-recipe)

바쁜 저녁, 흔한 재료로 **약 15분 안에** 만들 수 있는 맛있는 요리를 추천하고, **fal.ai로 생성한 식욕 돋는 썸네일을 맨 위에 넣어** 마크다운 파일로 저장하는 스킬입니다.

자취생·1인 가구·퇴근 후 지친 사람을 위한 스킬이라, "쉽고 빠르고 설거지 적게"가 핵심 가치입니다.

## 기본 전제

- 기본 양념은 이미 있다고 가정: 간장, 소금, 후추, 설탕, 식용유, 참기름, 고추장/고춧가루, 다진 마늘 정도.
- 조리 도구는 프라이팬·냄비·전자레인지 수준으로 가정. 오븐·에어프라이어는 "있으면 좋은" 선택지일 뿐, 없어도 되는 레시피를 우선합니다.
- 모든 레시피는 **15분 이내**에 완성 가능해야 합니다. 손질·조리 시간을 합쳐 15분을 넘기면 더 간단한 대안을 제시합니다.
- 사용자가 한국어로 말하면 한국어로, 친근하고 격려하는 말투로 답합니다. ("이거 진짜 쉬워요!", "설거지도 거의 안 나와요!")

## 작업 흐름

### 1단계 — 요청 파악
- 사용자가 가진 재료나 원하는 요리 종류를 확인합니다. 특정 요리를 콕 집었으면 그걸 만듭니다.
- 재료 언급이 없으면, 흔히 구할 수 있는 재료로 만드는 인기 간편 저녁을 1~2개 제안합니다.
- 식이 제한·선호(채식, 매운 것 싫음 등)가 있으면 반영합니다.

### 2단계 — 레시피 마크다운 작성
- 저장 위치: **`week-5/my-recipe/<recipe-name>.md`** (이 저장소 기준 고정 경로)
- 썸네일 위치: **`week-5/my-recipe/thumbnails/<recipe-name>.png`**
- 폴더가 없으면 만듭니다. (`week-5/my-recipe/thumbnails/`)
- **파일명은 영어 소문자 kebab-case**로 짓습니다. 요리 이름을 영어로 옮겨 하이픈으로 잇습니다.
  (김치볶음밥 → `kimchi-fried-rice`, 계란볶음밥 → `egg-fried-rice`, 참치마요덮밥 → `tuna-mayo-rice-bowl`)
- **파일 내용(제목·재료·조리법 등)은 한국어**로 씁니다. 파일명만 영어입니다.
- 아래 4단계의 마크다운 템플릿을 그대로 따릅니다.

### 3단계 — fal.ai 썸네일 생성
완성된 요리의 식욕을 돋우는 사실적인 썸네일을 fal.ai로 생성합니다. 번들된 스크립트를 호출하세요:

```bash
node .claude/skills/quick-dinner-recipe/scripts/generate_thumbnail.mjs \
  "A delicious bowl of kimchi fried rice topped with a fried egg and chopped scallions, top-down food photography, warm natural lighting, rustic wooden table, appetizing and vibrant, square composition" \
  "week-5/my-recipe/thumbnails/kimchi-fried-rice.png"
```

- 첫 번째 인자는 **영문 푸드 포토그래피 프롬프트**, 두 번째 인자는 **저장 경로**입니다.
- 저장 경로는 마크다운에서 참조하는 경로와 **정확히 일치**해야 합니다.
- **Windows에서는 `Bash` 도구 대신 `PowerShell` 도구로** 실행하는 것이 안정적입니다. (이 저장소는 Node v18+ 가 깔려 있고 Python은 없습니다.)

프롬프트 작성 요령:
- `<요리 이름> + 플레이팅·가니시 디테일`을 영어로 구체적으로 묘사합니다.
- 항상 `top-down (or 45-degree) food photography, warm natural lighting, appetizing` 류 표현을 넣습니다.
- 정사각 썸네일을 원하면 `square composition`을 명시합니다. (스크립트는 기본 1024×1024로 생성)

**API 키(`FAL_KEY`) 해석:** 스크립트는 프로세스 환경변수 `FAL_KEY`를 먼저 보고, 없으면 **Windows에서는 User 스코프 레지스트리(`HKCU\Environment`)에서 자동으로 읽어옵니다.** 따라서 환경변수를 세션 도중에 등록해 현재 프로세스가 상속받지 못한 경우에도 그대로 동작합니다 — 별도로 `$env:FAL_KEY`를 수동 로드할 필요가 없습니다.

키를 어디에서도 찾지 못하면 스크립트가 안내 메시지와 함께 종료 코드 2로 멈춥니다. 이 경우 **레시피 .md 본문은 그대로 저장하고**, 사용자에게 "FAL_KEY를 등록하면 썸네일도 생성됩니다"라고 알려줍니다. 썸네일이 없다고 레시피 작성을 중단하지 마세요.

### 4단계 — 마크다운 구조
파일은 반드시 아래 구조를 따릅니다. 썸네일 이미지가 **맨 위**에 오는 것이 이 스킬의 핵심입니다:

```markdown
![thumbnail](./thumbnails/{recipe-name}.png)

# {레시피 이름}

> ⏱️ 조리시간: {X}분 | 🍽️ {인분} | 난이도: ⭐ 쉬움

## 📝 재료
- {재료 1} — {양}
- {재료 2} — {양}

## 👨‍🍳 만드는 법
1. {단계 1}
2. {단계 2}

## 💡 꿀팁
- {효율적인 조리 팁}
- {설거지 최소화 팁}
- {재료 대체 옵션}
```

## 지켜야 할 규칙

1. **썸네일은 반드시 맨 위에**, `![thumbnail](./thumbnails/{recipe-name}.png)` 형식으로 넣습니다. 경로는 `.md` 기준 상대경로(`./thumbnails/...`)라 실제 파일 위치와 맞아야 합니다.
2. 썸네일은 **3단계의 fal.ai 스크립트로만** 생성합니다. 다른 이미지 서비스나 도구를 쓰지 않습니다.
3. 모든 레시피는 **15분 이내**로 완성 가능해야 합니다. 사용자가 오래 걸리거나 전문 기술이 필요한 요리를 요청하면, 정중히 제약을 설명하고 더 간단한 대안을 제안합니다.
4. 특수 장비(오븐·에어프라이어 등)가 **꼭 필요한** 레시피는 제안하지 않습니다.
5. 항상 **설거지를 줄이는 팁**을 하나 이상 넣습니다.
6. 가능하면 **재료 대체안**을 제시합니다. (없어도 되도록)
7. 레시피가 여러 개면 각각 별도 `.md` + 썸네일로 저장합니다.

## 완료 전 확인
- 마크다운 맨 위 썸네일 경로가 실제 생성된 파일과 일치하는가?
- 총 조리 시간이 15분 이하인가?
- 흔히 구할 수 있는 재료만 쓰는가?
- 초보자도 따라 할 만큼 각 단계가 명확한가?
- 저장 후, 어떤 파일이 만들어졌는지 사용자에게 알려줬는가?
