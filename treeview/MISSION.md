# 🎯 APP MISSION

## Project Name
**treeview** (확정)

## Mission Statement
두서없이 쏟아낸 생각과 긴 글·문서를 넣으면, 핵심을 요약하고 조각들의 연관성을 찾아 **한 장의 마인드맵**으로 정리해주는 — 산만한 사고를 위한 개인용 생각정리 도구.

## Problem Statement
생각이 많고 산만한 사람은 머릿속 아이디어가 파편적으로 흩어져 있어, 핵심이 무엇이고 어디로 나아가야 할지 스스로 구조화하기 어렵다. 기존 메모/마인드맵 도구는 "직접 정리"를 요구하기 때문에, 정작 정리가 필요한 순간에는 오히려 부담이 된다. 이 앱은 정리 작업 자체를 앱에 위임하여, 복잡한 생각을 넣기만 하면 요약과 지도가 자동으로 나오게 한다.

## Target Users
- **1차(현재):** 만든 사람 본인 — 생각이 많고 산만해서, 아이디어를 외부에 맡겨 구조화하고 싶은 사람.
- **2차(향후):** 같은 불편을 겪는 일반 사용자 — 지식 노동자, 기획자, 학생, 글 쓰는 사람 등 "머릿속 정리"가 잦은 사람. 완성도가 확보되면 광고 없이 무료+부분 유료 모델로 공개 배포.

## App Type
- [x] Personal/Private Use (우선)
- [x] Public/Commercial (완성도 확보 후 배포 목표)

## Core User Scenario (성공의 순간)
> 머릿속이 복잡한 날, 앱을 열어 두서없이 쏟아낸 생각(또는 읽던 기사·PDF·회의록)을 큰 입력창에 붙여넣고 버튼 하나를 누른다. 몇 십 초 안에 **핵심만 딱딱 뽑은 요약**과, 그것을 한눈에 보여주는 **깔끔한 마인드맵**이 뜬다. 뒤엉켰던 생각이 한 장의 지도로 정리되어, 어디로 나아가야 할지 방향이 보이고 마음이 가벼워진다.

## Output Representation (결과물 표현 방식)
텍스트를 넣으면 아래 형태로 결과를 보여준다. 형태는 옵션으로 전환 가능.
- **요약 + 맵 (기본):** 짧은 핵심 요약 문단 + 그 아래 마인드맵.
- **트리형:** 중심 주제에서 가지가 뻗어나가는 위계형 지도.
- **그물망/클러스터형 (선택):** 관련 개념끼리 색깔 덩어리로 묶고, 중요한 개념일수록 노드를 크게 표시.
- **아웃라인 ↔ 맵 토글:** 같은 내용을 접었다 폈다 하는 텍스트 개요와 마인드맵을 버튼 하나로 전환(글로 훑고 → 맵으로 구조 확인).
- **내보내기:** PNG / Markdown 형식으로 결과물 저장.

## Core Features — v1 (무료)
1. **입력:** 긴 텍스트·메모 붙여넣기, 외부 문서(PDF·기사) 불러오기.
2. **핵심 요약 (연관성 범위 a):** 넣은 내용 "안에서" 핵심 정리 + 조각 간 관계 파악.
3. **마인드맵 시각화:** 요약+맵(기본) / 트리형 / 그물망형(선택) / 아웃라인 토글.
4. **내보내기:** PNG · Markdown.
5. **모바일 최적화 (1급 요구사항):** 스마트폰 사용 비중이 높은 만큼, 모바일 우선(mobile-first)·반응형으로 설계. 작은 화면에서도 요약과 마인드맵이 잘 읽히고, 터치로 편하게 조작(핀치 줌·드래그·탭)할 수 있어야 함. → [세부 사항은 아래 Mobile-First Requirement 참고]
> 한 줄 요약: **"넣으면 → 핵심 요약 + 한 장의 지도"** 를, **폰에서도 데스크톱에서도 매끄럽게**. 까지가 v1.

## Mobile-First Requirement (v1 핵심)
많은 사용자가 스마트폰으로 접근한다는 전제를 v1의 1급 요구사항으로 못 박는다.
- **모바일 우선 반응형:** 작은 화면을 기준으로 먼저 설계하고 데스크톱으로 확장. 가로 스크롤·잘림 없이 화면 폭에 맞춰 재배치.
- **터치 친화적 조작:** 마인드맵은 핀치 줌 / 드래그 이동 / 노드 탭으로 펼치기·접기. 버튼·탭 영역은 손가락에 맞는 충분한 크기.
- **작은 화면에서의 가독성:** 요약은 접었다 펼 수 있게, 맵은 자동 맞춤(fit-to-screen) + 노드 겹침 최소화. 필요 시 요약↔맵을 탭으로 전환.
- **폰에서 붙여넣기·문서 열기 흐름이 매끄러워야** 한다(모바일에서 텍스트 붙여넣기, 파일 선택이 번거롭지 않게).

## Premium Features — v2 (유료)
1. **새 관점·빠진 고리 제안 (연관성 범위 b):** 넣은 내용을 넘어, 묶이는 상위 주제나 놓친 연결을 능동 제안.
2. **과거 생각과의 연결 (연관성 범위 c):** 이전에 넣었던 메모·맵과 연관성을 찾아 이어줌.
3. **(후보) 웹검색 기반 아이디어 확장:** 관련 자료·관점을 외부에서 찾아 추천.

## Free / Paid Boundary (경계 모델)
- **기능은 전부 무료로 개방**하되, **월 생성 횟수(요약/맵 생성)에 상한**을 둔다.
- **무제한 사용 + v2 심화 기능(b·c)**을 **유료 구독**으로 제공.
- 광고는 붙이지 않는다 — "머리를 비우려는" 집중 사용 맥락과 충돌하기 때문.
- (검토안) 첫 며칠은 프리미엄 전체를 열어주는 리버스 트라이얼로 심화 기능의 가치를 먼저 체험시키는 방식.

## Simple UI Direction
"UI는 최대한 단순하게" + "모바일에서도 최적화"라는 요구를 반영한 방향(리서치 기반):
- **단일 입력창 중심의 미니멀 레이아웃:** 화면 중앙에 큰 입력창 하나 + 생성 버튼. 첫 화면에서 "붙여넣고 누르면 끝"이 되도록 한다. (폰에서도 동일하게 성립)
- **여백으로 구분, 선·박스 최소화:** 구획을 나눌 때 경계선 대신 여백/옅은 배경 톤을 사용해 시각적 소음을 줄인다.
- **맵 목록은 화면 크기에 따라 적응:** 데스크톱에서는 접이식 좌측 사이드바로, 모바일에서는 하단 시트나 햄버거 메뉴 등으로 이전 맵 목록을 최소한만 제공. 극단적 미니멀을 해치지 않게.
- **노드는 키워드·짧은 문구 위주, 방해 제로:** 메뉴를 파고들지 않아도 결과를 바로 읽을 수 있게. (참고: Xmind의 'Zen Mode'식 몰입형 화면)
- **터치·모바일 우선:** 위 Mobile-First Requirement를 UI 전반의 기본 전제로 삼는다(터치 조작, 반응형 재배치, 작은 화면 가독성).

## Success Metrics
| Metric | Target | Timeframe |
|--------|--------|-----------|
| 붙여넣기 → 맵 생성까지 체감 시간 | 30초 이내 | v1 출시 시 |
| 모바일에서 요약+맵 생성·열람 성공률 (주요 폰 화면폭에서 잘림·오작동 없이) | 95% 이상 | v1 출시 시 |
